# Identity Model

---

> **Understand. Remember. Reason. Evolve.**

---

# Purpose

Identity is the central self-model of AlexOS.

While Long-Term Memory stores events, Identity stores understanding.

Memory answers:

> What happened?

Identity answers:

> Who is this person?

Identity continuously evolves as AlexOS learns from interactions.

---

# Identity Architecture

```
Identity
│
├── Basic Profile
│
├── Preferences
│
├── Goals
│
├── Projects
│
├── Knowledge
│
├── Skills
│
├── Relationships
│
├── Decisions
│
├── Values
│
├── Timeline
│
└── Statistics
```

---

# Basic Profile

Stores stable information.

Examples:

- Name
- Preferred language
- Timezone
- Country
- Occupation

---

# Preferences

Stores user preferences.

Examples:

- Linux
- Python
- Hallberg-Rassy
- Local AI
- Privacy

Preferences may change over time.

---

# Goals

Stores active goals.

Examples:

- Build AlexOS
- Learn AI
- Sail around the world

Goals have priorities.

Goals may be completed.

---

# Projects

Stores long-running projects.

Examples:

- AlexOS
- Meditation App
- Sailing Route Planner

Projects contain milestones.

---

# Knowledge

Represents what AlexOS believes the user already knows.

Examples:

- Python
- Git
- Docker
- SQL

Confidence increases over time.

---

# Skills

Represents practical abilities.

Examples:

- Programming
- Data Analysis
- Sailing

Each skill has a confidence level.

---

# Relationships

Stores important people and entities.

Examples:

Family

Friends

Companies

Organizations

---

# Decisions

Stores important decisions.

Examples:

Use Ollama

Choose ChromaDB

Use GitHub

---

# Values

Stores long-term priorities.

Examples:

Privacy

Reliability

Engineering Quality

Learning

---

# Timeline

Chronological summary of major life events.

---

# Statistics

Automatically calculated values.

Examples:

Total conversations

Preferences discovered

Goals completed

Projects

Knowledge growth

---

# Identity Evolution

Identity is never edited manually.

Identity evolves only through Cognitive Events.

Perception

↓

Memory Extraction

↓

Reflection

↓

Long-Term Memory

↓

Identity Update

---

# Guiding Principle

Memory stores experiences.

Identity stores understanding.

AlexOS must protect Identity as its most valuable asset.
